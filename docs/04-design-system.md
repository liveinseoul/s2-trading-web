# 04 · 디자인 시스템

깔끔·정보 밀도 높은 금융 대시보드. **모바일 우선**, Tailwind CSS v4.

## 1. 색상 — ⚠️ 한국식 관례 (빨강=상승/이익, 파랑=하락/손실)

미국과 반대다. 한국 주식 이용자는 **빨강=오름·매수·이익, 파랑=내림·손실**로 인식한다. 반드시 이 관례를 따른다.

```
상승/이익/매수   up      #E03131 (red-600 계열)   배경 연: #FFF5F5
하락/손실/매도   down    #1971C2 (blue-700 계열)  배경 연: #E7F5FF
중립/보합        flat    #495057 (gray-600)
```

기타 토큰
```
배경    bg        #FFFFFF / 다크 #0B0E14
표면    surface   #F8F9FA / 다크 #151A23 (카드)
경계    border    #E9ECEF / 다크 #232A36
본문    text      #212529 / 다크 #E9ECEF
보조    muted     #868E96
강조    accent    #1098AD (teal, 링크/버튼 — 상승/하락 색과 충돌 안 하게 청록 계열)
경고    warn      #F08C00 (레버 미체결·주의)
```

손익 표시 헬퍼: 값 ≥ 0 → `text-up`, < 0 → `text-down`. 부호(+/−) 항상 표기.

## 2. 타이포그래피

- 본문 폰트: **Pretendard**(한글 가독·숫자 정렬 우수) → 폴백 system-ui.
- 숫자(가격·수익률·수량)는 **tabular-nums**(`font-variant-numeric: tabular-nums`)로 자릿수 정렬.
- 스케일: 페이지 제목 20–24px / 섹션 16–18px / 본문 14px / 캡션 12px. 모바일에서 1단계 축소.

## 3. 레이아웃 · 반응형

- 모바일(<640): 1열 카드 스택 + **하단 고정 탭바**(오늘·대시보드·종목·규칙).
- 태블릿(≥768): 2열. 데스크톱(≥1024): 콘텐츠 max-w-5xl 중앙 + 상단 네비.
- **표→카드 전환**: 보유/체결/거래 표는 좁은 화면에서 행을 카드로 접는다(종목명+핵심수치 2~3줄).
  넓은 화면은 정렬된 테이블(tabular-nums).

## 4. 핵심 컴포넌트

| 컴포넌트 | 설명 |
|---|---|
| `SummaryBar` | NAV·레버리지·보유수·최종 업데이트. 모바일 가로 스크롤 칩. |
| `StatCard` | 라벨 + 큰 수치 + 부가(증감). 손익색 적용. |
| `OrderCard` | 동시호가 주문 1건: 종목·주문가·포트%·도달배지. 환산수량(자본입력 시). |
| `ExecutionRow/Section` | 체결 분류(신규/추가/매도/손절/미체결). action별 배지·색. |
| `PositionList` | 보유 종목 카드/행. 평가손익 색. |
| `NavChart` | NAV 라인 + DD 영역. 클라이언트(recharts 또는 visx). |
| `MonthlyHeatmap` | 월×수익률 히트맵(빨강↑/파랑↓ 농담). |
| `TradeTimeline` | 종목 상세: 진입→추가→매도 타임라인. |
| `Badge` | 시장(KS/KQ), MA120(UP/DOWN), 양봉/음봉, 도달(●). |
| `CapitalInput` | 자본 입력 → localStorage → 환산 컨텍스트. |
| `DisclaimerBanner` | 면책 고지(상시). |

배지 규칙: 시장 KS/KQ(중립 회색), MA120 UP(빨강 톤)/DOWN(파랑 톤), 양봉(빨강)/음봉(파랑), 레버 미체결(warn).

## 5. 숫자·표기 규칙

- 가격: 천단위 콤마, 원 단위(`12,345`). 수익률: `+7.0%`/`−3.2%`(부호+색).
- 종목명: **6자 초과 시 앞 6자**(엔진 출력 규칙과 일치). 전체명은 툴팁/상세에서.
- 큰 금액: 억/만 단위 보조 표기 옵션(`1.27억`).
- 날짜: `YYYY-MM-DD`, 요일 보조.

## 6. 접근성·기타

- 색만으로 정보 전달 금지 → 부호/아이콘 병기(색각 보조).
- 다크 모드: 토큰 다크 값 제공(Phase 2, `prefers-color-scheme`).
- 모든 표 헤더·배지에 의미 라벨. 면책은 닫아도 푸터에 상시 노출.
