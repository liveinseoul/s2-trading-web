# 06 · 주문 라이프사이클 & 감시주문 플랜

S2 주문은 **상태 의존적**이다(체결이 다음 주문을 바꾼다). 이 문서가 "어떤 감시주문을 언제 걸고/바꾸는가"를 정의한다.

## 0. 대전제 — 모델 포트폴리오 ≠ 사용자 계좌

서비스는 **단일 기준 모델 포트폴리오**의 상태를 KRX 시세로 계산할 뿐, 사용자 증권계좌와 연동되지 않는다.
따라서 모든 안내는 *"모델과 동일하게 매매하려면 지금 이 주문을 거세요"*이며, 실제 체결·정정은 사용자 브로커에서 한다.
(알림도 자동매매가 아니라 수동 설정 안내.)

## 1. 주문 유형과 설정 시점

| # | 주문 | 방식 | 가격 | 언제 설정 |
|---|---|---|---|---|
| 1 | **1차 매수** | 마감 **동시호가** 지정가 | 지지선(MA20×0.80) | 당일 15:20~15:30 (사전 후보 = `daily_candidates`) |
| 2 | **2·3차 매수** | **감시(조건부)** 매수 | 직전 매수가 × 0.9 | 직전 회차 체결 후(그날 저녁)부터 상시 |
| 3 | **분할매도 1/2/3** | **감시** 매도 (10/10/80) | 평단 ×1.03/1.05/1.07 | 매수 보유 중 상시 — **평단 바뀌면 재계산** |
| 4 | **손절** | **감시** 매도 (잔량 전량) | 직전 매도단계가(당일 −1% 버퍼) | **분할매도 한 단계 체결 후** 설정 |
| 4b | **신저가 손절** | 감시 매도 (2차 매수 후) | 직전 최저가 하향 → 종가청산 | 2차 매수 후, 저가 갱신 추적 |

## 2. 상태 의존성 — "재설정"이 필요한 이유

```
1차 매수 체결 ─► 매도 감시(평단=1차가 ×1.03/1.05/1.07) + 2차 매수 감시(1차가×0.9) 설정
        │
        ├─(2차 매수 체결)─► 평단 ↓ ─► ★매도 감시 3건 모두 재계산(낮아짐) + 3차 매수 감시(2차가×0.9)
        │                              + (신저가 손절 감시 활성)
        │
        └─(+3% 매도 체결)──► ★손절 감시 신규 설정(손절가=평단×1.03, 당일 −1% 버퍼) + 매도 감시는 +5/+7만 잔존
                                   └─(되밀려 손절가 이탈)─► 잔량 전량 청산
```

★ 표시가 기존 설계에 안내되지 않던 누락 지점(사용자 지적): **2차 매수 후 매도 재설정 / 매도 후 손절 설정**.

## 3. 해법 A — 장마감 후 "일일 감시주문 플랜" (Phase 1, 결정적)

매일 마감 후, 모델의 당일 체결을 반영해 **다음 거래일에 세팅할 감시주문 전체 세트**를 산출·발행한다.
EOD 데이터로 100% 결정되므로 실시간 불필요. **§2의 재설정 문제를 결정적으로 해소.**

- 보유 종목별 출력:
  - 매수 감시(2/3차): 가격(직전매수가×0.9)·수량·포트%
  - 매도 감시(미체결 단계): 단계(+3/+5/+7%)·목표가·수량
  - 손절 감시: 가격(매도단계가)·수량 / 신저가 손절: 기준 최저가
- **전일 대비 diff** 표기: `new`(신규 설정) / `changed`(가격·수량 변경) / `keep`(유지) / `cancel`(취소).
- 사용자는 저녁에 이 세트대로 브로커 감시주문을 갱신.
- 데이터: `daily_order_plan` (docs/02 참조).

## 4. 해법 A의 한계 — 당일(intraday) 시차

저녁 리셋만으로는 **당일 발생→당일 대응** 이벤트를 못 따라간다:
- **당일 −1% 손절**: 1차 매도(+3%)가 장중 체결되면 같은 날 손절선이 생기고, 되밀리면 같은 날 청산.
  백테스트상 **1차 매도 발생일의 약 49%가 같은 날 손절** → 비중이 커서 추적 충실도에 영향.
- 2차 매수 장중 체결 후 낮아진 매도 목표를 그날 장중 반영 불가.
→ 이 시차는 화면에 명시(과대평가 방지)하고, 해법 B로 닫는다.

## 5. 해법 B — 장중 모니터링 + 텔레그램 알림 (Phase 2/3, 권장)

모델을 KRX 시세로 장중 주기 감시 → **모델 트리거 교차 시** "지금 이 감시주문을 설정/변경하세요" 푸시.
브로커 연동 불필요(모델 추적, 계좌 아님).

- 알림 예시
  - *"[로보티즈] 모델 2차 매수가 186,750 도달 → 체결 시 매도감시 +3/+5/+7%=A/B/C, 손절 D로 재설정"*
  - *"[에스피지] 모델 +3% 매도 체결 → 잔량 손절 감시 E원 설정"*
  - *"[OOO] 모델 신저가 손절 임박(직전 최저가 F 근접)"*
- 감시 주기: 단순 시간당도 가능하나 **트리거 근접/교차 이벤트 기반**이 효율적. **마감 임박(15:00~15:20) 집중 점검** 권장.
- 채널: 텔레그램 봇(무료·간편) 우선. 추후 웹푸시/이메일.
- 한계: 자동매매 아님(수동 설정 안내). 모델 체결 ≠ 사용자 실제 체결.

| Phase 1 | 일일 감시주문 플랜(저녁) | 마감 후 | 2차후 매도재설정·매도후 손절설정(결정적) |
