# 03 · 화면(라우트) 명세

App Router 기준. 모든 화면은 **모바일 우선**. 서버 컴포넌트에서 Supabase 조회 → 렌더, 클라이언트 컴포넌트는
자본입력 환산·날짜 선택·차트 등 상호작용에만 사용.

## 라우트 맵

```
app/
├─ layout.tsx              # 공통: 헤더, 하단 탭(모바일), 면책 배너, 자본설정 컨텍스트
├─ page.tsx                # /            오늘 (홈)
├─ day/[date]/page.tsx     # /day/2026-03-04   일자별 매매
├─ dashboard/page.tsx      # /dashboard   월별 대시보드
├─ stocks/page.tsx         # /stocks      종목 리스트
├─ stocks/[ticker]/page.tsx# /stocks/108490    종목 상세
├─ trades/page.tsx         # /trades      전체 거래내역
└─ rules/page.tsx          # /rules       S2 규칙 설명
```

모바일 하단 탭: **오늘 · 대시보드 · 종목 · 규칙** (4탭). 일자별/거래내역은 홈·대시보드에서 진입.

---

## 1. `/` 오늘 (홈) — 핵심 화면

장마감 전엔 "오늘 낼 주문", 장마감 후엔 "오늘 체결 결과 + 보유"를 보여준다.

**구성**
1. **상단 요약 바** — NAV, 레버리지, 보유 종목수, 최종 업데이트 시각(`meta.last_*`). 면책 한 줄.
2. **동시호가 주문 (장중 15:10~15:30, Phase 2)** — `daily_candidates`
   - [A] 신규 진입: 종목명·시장(KS/KQ)·현재가·**주문가(지지선)**·포트%·MA120(UP/DOWN)·직전스파이크(양/음봉)·지지선까지(%)
   - [B] 추가매수: 종목명·회차·평단·현재가·**주문가(직전매수가×0.9)**·포트%·도달여부
   - 각 행에 "내 자본 기준 수량" 환산값(자본 입력 시).
   - 안내: *15:20~15:30 동시호가에 지정가로 미리 주문. 종가가 주문가 이하면 체결, 아니면 미체결(위험 없음).*
3. **오늘 체결 결과 (장마감 후)** — `executions` (당일)
   - [A] 신규 체결 / [B] 추가매수 체결 / [C] 레버 한도 미체결(참고, 흐리게)
4. **오늘 저녁 세팅할 감시주문 플랜** — `daily_order_plan` (최신일) — *핵심: 누락됐던 매도·손절 안내*
   - 보유 종목별: [2/3차 매수 감시] [매도 감시(+3/+5/+7%, 미체결 단계)] [손절 감시(매도단계가)] / 신저가 손절 기준.
   - 전일 대비 **신규/변경/취소** 배지로 "오늘 바꿔야 할 것"만 빠르게 식별.
   - 안내: *체결로 평단·단계가 바뀌면 매도·손절 가격이 재계산됨 → 저녁에 이 세트로 갱신.* (라이프사이클: docs/06)
5. **현재 보유 포트폴리오** — `position_snapshots` (최신일)
   - 종목·진입일·매수/매도차수·평단·현재가·평가손익·수익률·포트%. 모바일은 카드.

**데이터**: `meta`, `daily_candidates(d=today)`, `daily_order_plan(d=latest)`, `executions(d=latest)`, `position_snapshots(d=latest)`
**컴포넌트**: `SummaryBar`, `OrderCard`(신규/추가), `WatchOrderPlan`(매수/매도/손절+diff배지), `ExecutionTable`, `PositionList`, `CapitalInput`

## 2. `/day/[date]` 일자별 매매

특정 일자에 매수/매도했어야 한 종목 목록(검증·복기용).

**구성**
- 날짜 선택기(달력 또는 ◀ 이전/다음 거래일 ▶).
- **신규 진입** / **추가매수** / **매도·손절** 섹션 — `executions(d)` 를 action별 분류.
  각 종목: 가격·수량·금액·포트%·MA120·직전스파이크. 매도는 단계(+3/+5/+7%)·손절 사유.
- **레버 한도 미체결(참고)** — `blocked_by_leverage=true`, 미체결 금액 합계.
- **그날 마감 후 보유** — `position_snapshots(d)`.

**데이터**: `executions(d)`, `position_snapshots(d)`, `nav_daily(d)`
**컴포넌트**: `DatePicker`, `ExecutionSection`, `PositionList`

---

## 3. `/dashboard` 월별 대시보드

룰을 따랐을 때의 월별 성과.

**구성**
1. **누적 성과 카드** — 전 구간 CAGR·MDD·Calmar·총 거래수·승률(검증 수치 병기: 7년 vs 12년 정직 구간).
2. **NAV 곡선 + 낙폭(DD)** — `nav_daily` 라인 차트(기간 토글: 1Y/3Y/전체).
3. **월별 표/히트맵** — `monthly_stats`: 월별 수익률 히트맵(빨강=플러스/파랑=마이너스), 매매건수·승률·평균수익률·실현손익·MDD.
4. **월 상세 드릴다운** — 월 클릭 → 해당 월 거래 목록(`trades` where exit in month) + `/day` 링크.

**데이터**: `monthly_stats`, `nav_daily`, `trades`
**컴포넌트**: `StatCards`, `NavChart`(클라이언트, recharts/visx), `MonthlyHeatmap`, `MonthlyTable`

## 4. `/stocks` 종목 리스트 · `/stocks/[ticker]` 종목 상세

**리스트**: 거래가 있었던 종목 검색/필터(시장·기간·승패). 종목별 거래수·누적손익·평균수익률·최근거래일.
**상세**: 해당 종목의 모든 거래(`trades`)와 레그(`trade_legs`)
- 거래별 타임라인: 진입일→추가매수(일자·가격·포트%)→분할매도/손절(일자·가격) → 수익률·손익.
- 일자·포트% 포함된 매수/매도 표 (요청 사항 충족).
- 종목 가격 + 매매 시점 마커 차트(선택, Phase 2).

**데이터**: `trades`, `trade_legs`
**컴포넌트**: `StockSearch`, `TradeTimeline`, `LegTable`

## 5. `/trades` 전체 거래내역

완결 거래 전체를 필터·정렬(기간·시장·수익률·보유일). CSV 내보내기(선택).
**데이터**: `trades` (+ 행 클릭 → `/stocks/[ticker]`)

## 6. `/rules` S2 규칙 설명

따라하려는 사람을 위한 규칙·근거 교육. 정적 콘텐츠 + `meta.rules_version`.
- 진입: 거래대금 ≥5,000억 스파이크(직전 60일) + 종가 < 20일선 −20% + 거래대금 리셋
- 사이징: MA120 위 15% / 아래 7.5%, 직전 스파이크 음봉이면 ×0.8(12%/6%)
- 추가매수: 직전 매수가 −10%, 최대 3차, 1차와 동일액
- 분할매도: 평단 +3/+5/+7% → 10/10/80, 손절 상향
- 손절: 신저가 손절(2차 후), 매도 당일 −1% 버퍼(현실 모델)
- 레버리지 1.3배 상한, 기준자본 5억
- 주문 실무: 신규=지지선 지정가 동시호가, 추가매수=직전매수가×0.9 지정가
- **검증 성과·한계**: 7년 Calmar ~2.0 / 12년 시점정확 Calmar ~0.7(생존편향·기간효과 정직 공개), 슬리피지·용량 한계

## 공통 요소

- **면책 배너**(전 화면 상단 또는 푸터): "투자 정보·교육용, 투자자문 아님, 과거성과≠미래, 손익 본인 책임."
- **자본 입력 위젯**: 헤더에서 자본 입력 → `localStorage` 저장 → 전 화면의 포트%를 수량/금액으로 환산(클라이언트).
- **최종 업데이트**: `meta.last_eod_at` / `last_preclose_at` 표시(데이터 신선도).
- **로딩/빈 상태**: 거래 없는 날·휴장일 명확히 표시.
